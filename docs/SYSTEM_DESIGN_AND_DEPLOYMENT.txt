================================================================================
                          1. PROJECT OVERVIEW
================================================================================

This is a full-stack Task Management Application inspired by JIRA and Google 
Task Tracker. It provides comprehensive project and task management capabilities 
with modern features including:

- Task CRUD operations with Kanban board visualization
- Real-time collaboration with live cursors and presence indicators
- AI-powered code suggestions and task analysis
- Gamification with XP, levels, and leaderboards
- Smart dependency management with AI suggestions
- Analytics dashboard for managers
- Role-based access control (Developer, PM, PO)

================================================================================
                      4. BACKEND DESIGN & LOGIC
================================================================================

4.1 CORE MODULES:
-----------------

main.py (API Entry Point)
- Defines all REST API endpoints
- Configures CORS middleware for cross-origin requests
- Mounts static file serving for uploads
- Implements WebSocket endpoint for real-time features
- Route categories: Auth, Users, Projects, Tasks, Comments, Analytics, AI

auth.py (Authentication & Authorization)
- JWT token-based authentication
- Password hashing with bcrypt
- Token expiration handling (30 minutes default)
- Role-based access control decorators:
  * get_current_user_json: Any authenticated user
  * require_manager_role: PM/PO only access

json_storage.py (Data Persistence)
- File-based JSON storage for simplicity
- Atomic read/write operations
- Data structure: users, projects, tasks, comments, gamification

schemas.py (Data Validation)
- Pydantic models for request/response validation
- User, Project, Task, Comment schemas
- Automatic validation and serialization

4.2 SERVICE MODULES:
--------------------

ai_assistant.py (AI Pair Programming)
- Pattern-based code suggestions
- Task complexity estimation using keyword analysis
- Subtask generation for complex tasks
- Code hints for common patterns (JWT, React, Testing)
- Algorithm: O(k) keyword matching using Set operations

ai_insights.py (AI Analytics)
- Predictive sprint completion analysis
- Resource optimization suggestions
- Bottleneck detection
- Trend analysis for productivity

collaboration_service.py (Real-time Collaboration)
- WebSocket connection management
- Presence tracking (who's online)
- Live cursor sharing
- Typing indicators
- Task edit locking to prevent conflicts
- Broadcast messaging to task viewers

dependency_service.py (Smart Dependencies)
- Task dependency graph management
- Circular dependency detection (DFS algorithm)
- AI-based dependency suggestions
- Blocked task identification
- Graph visualization data generation

kpi_service.py (KPI & Analytics)
- Task completion velocity calculation
- Team performance metrics
- Risk detection algorithms
- Sprint health scoring
- Individual developer statistics

gamification_service.py (Gamification Engine)
- XP calculation and awarding
- Level progression system
- Achievement tracking
- Leaderboard generation
- Streak tracking

codebase_rag_service.py (Codebase Search)
- Code file indexing
- Vector embedding generation
- Semantic similarity search
- RAG-based code Q&A

================================================================================
                   7. AUTHENTICATION & AUTHORIZATION
================================================================================

7.1 AUTHENTICATION FLOW:
------------------------
1. User submits credentials (username/password)
2. Backend validates against hashed password (bcrypt)
3. If valid, generates JWT token with:
   - user_id
   - username
   - role
   - expiration time (30 minutes)
4. Token returned to client
5. Client stores token in localStorage
6. All subsequent requests include token in Authorization header

7.2 AUTHORIZATION LEVELS:
-------------------------

DEVELOPER:
- Create/Update/View tasks
- Add comments
- Use AI assistant features
- View personal gamification stats

PROJECT MANAGER (PM):
- All developer permissions
- Access analytics dashboard
- View team KPIs
- Create sprints

PRODUCT OWNER (PO):
- All PM permissions
- Full project access
- Risk assessment access

================================================================================
                  8. REAL-TIME COLLABORATION SYSTEM
================================================================================

8.1 WEBSOCKET MESSAGE TYPES:
----------------------------

PRESENCE EVENTS:
- user_joined: New user came online
- user_left: User went offline
- presence_sync: Full presence state sync

CURSOR EVENTS:
- cursor_move: User cursor position update
- cursor_leave: User stopped viewing task

TASK EVENTS:
- task_update: Task was modified
- task_create: New task created
- task_delete: Task was deleted

LOCK EVENTS:
- lock_acquired: User started editing task
- lock_released: User finished editing
- lock_denied: Another user is editing

TYPING EVENTS:
- typing_start: User began typing
- typing_stop: User stopped typing

8.2 CONFLICT RESOLUTION:
------------------------
- Optimistic locking with task version tracking
- First-come-first-served edit lock acquisition
- 5-minute auto-release for stale locks
- Real-time update broadcasting to resolve conflicts

10.1 XP SYSTEM:
---------------
Actions and XP rewards:
- Create Task:      10 XP
- Complete Task:    25 XP + (story_points * 5)
- Add Comment:       5 XP
- Daily Login:      15 XP
- First Login:      50 XP
- Perfect Sprint:  100 XP bonus

10.2 LEVEL PROGRESSION:
-----------------------
Level = floor(sqrt(XP / 100))

Level | XP Required
------|-----------
1     | 100
2     | 400
3     | 900
4     | 1600
5     | 2500
...

10.3 ACHIEVEMENTS:

================================================================================
          12. DEPLOYMENT STEPS - PRODUCTION READY APPLICATION
================================================================================

STEP 1: PREREQUISITES
---------------------
Ensure you have installed:
- Python 3.9+
- Node.js 18+
- npm or yarn
- Git

STEP 2: DATABASE MIGRATION (Recommended for Production)
-------------------------------------------------------
Current: JSON file storage (data.json)
Recommended Migration Path:

Option A: PostgreSQL (Recommended)
1. Install PostgreSQL
2. Create database: CREATE DATABASE taskmanager;
3. Install SQLAlchemy: pip install sqlalchemy asyncpg
4. Update json_storage.py to use SQLAlchemy ORM
5. Run migrations: alembic upgrade head

Option B: MongoDB
1. Install MongoDB
2. Install motor: pip install motor
3. Replace JSON operations with MongoDB operations

STEP 3: ENVIRONMENT CONFIGURATION
---------------------------------
Create .env file in backend directory:

# Backend .env
DATABASE_URL=postgresql://user:password@localhost:5432/taskmanager
SECRET_KEY=your-very-long-and-secure-secret-key-here
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440
CORS_ORIGINS=https://yourdomain.com
ENVIRONMENT=production

# Optional: AI Services
OPENAI_API_KEY=sk-...  # For enhanced AI features
GOOGLE_API_KEY=...     # For Google embeddings

STEP 4: BACKEND DEPLOYMENT OPTIONS
----------------------------------

Option A: Traditional VPS (DigitalOcean, AWS EC2, etc.)
-------------------------------------------------------
1. SSH into server
2. Clone repository:
   git clone https://github.com/yourusername/task-manager.git
3. Install dependencies:
   cd backend
   pip install -r requirements.txt
4. Configure Gunicorn with Uvicorn workers:
   gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
5. Set up systemd service for auto-restart:
   
   /etc/systemd/system/taskmanager.service:
   [Unit]
   Description=Task Manager Backend
   After=network.target
   
   [Service]
   User=www-data
   WorkingDirectory=/path/to/backend
   ExecStart=/path/to/venv/bin/gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker
   Restart=always
   
   [Install]
   WantedBy=multi-user.target

6. Enable and start:
   sudo systemctl enable taskmanager
   sudo systemctl start taskmanager

Option B: Docker Deployment (Recommended)
-----------------------------------------
1. Create Dockerfile for backend:
   
   FROM python:3.11-slim
   WORKDIR /app
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   COPY . .
   CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

2. Create docker-compose.yml:
   
   version: '3.8'
   services:
     backend:
       build: ./backend
       ports:
         - "8000:8000"
       environment:
         - DATABASE_URL=postgresql://postgres:password@db:5432/taskmanager
       depends_on:
         - db
     
     frontend:
       build: ./frontend
       ports:
         - "80:80"
     
     db:
       image: postgres:15
       environment:
         - POSTGRES_PASSWORD=password
         - POSTGRES_DB=taskmanager
       volumes:
         - postgres_data:/var/lib/postgresql/data
   
   volumes:
     postgres_data:

3. Deploy:
   docker-compose up -d

Option C: Cloud Platform (Heroku, Railway, Render)
--------------------------------------------------
1. Create Procfile:
   web: uvicorn main:app --host 0.0.0.0 --port $PORT

2. Push to platform:
   heroku create task-manager-api
   git push heroku main

Option D: AWS/GCP/Azure Managed Services
----------------------------------------
1. Use AWS App Runner, GCP Cloud Run, or Azure Container Apps
2. Configure with container image
3. Set environment variables in console
4. Configure auto-scaling rules

STEP 5: FRONTEND DEPLOYMENT
---------------------------

1. Build production bundle:
   cd frontend
   npm install
   npm run build

2. Output is in /frontend/dist folder

Option A: Static Hosting (Netlify, Vercel, Cloudflare Pages)
------------------------------------------------------------
1. Connect GitHub repository
2. Set build command: npm run build
3. Set publish directory: dist
4. Add environment variable:
   VITE_API_URL=https://api.yourdomain.com

Option B: Nginx (Traditional VPS)
---------------------------------
1. Copy dist folder to server:
   scp -r dist/* user@server:/var/www/taskmanager

2. Configure Nginx:
   
   /etc/nginx/sites-available/taskmanager:
   server {
       listen 80;
       server_name yourdomain.com;
       root /var/www/taskmanager;
       
       location / {
           try_files $uri $uri/ /index.html;
       }
       
       location /api {
           proxy_pass http://localhost:8000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection "upgrade";
       }
       
       location /ws {
           proxy_pass http://localhost:8000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection "upgrade";
       }
   }

3. Enable site and restart:
   sudo ln -s /etc/nginx/sites-available/taskmanager /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx

STEP 6: SSL CERTIFICATE (Required for Production)
-------------------------------------------------
Using Let's Encrypt with Certbot:

   sudo apt install certbot python3-certbot-nginx

3. Auto-renewal is configured automatically
